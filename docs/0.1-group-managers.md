# 6 Group Manager (Tầng điều phối)

Mỗi nhóm A-F có **1 Group Manager** đảm nhiệm điều phối sub-agent + tổng hợp output. Group Manager là **entry point thật sự** của tầng nghiệp vụ — cả 2 đường (UI click + Supervisor route) đều converge ở đây.

## Mục đích chung

1. Classify message trong scope của nhóm → chọn 1 hoặc nhiều sub-agent
2. Chain (chuỗi) sub-agent khi cần nhiều bước
3. Tổng hợp output thành 1 response cohesive trả user
4. Ask back nếu câu hỏi ngoài scope nhóm

## 2 entry path

| Path | Cách Manager nhận message |
|---|---|
| **UI click (Mode 1-6)** | User chọn group card → message vào thẳng Manager (không qua Supervisor) |
| **Supervisor route (Mode 0)** | User free chat → Supervisor classify → route đến Manager |

Chi tiết 7 mode + conversation rule + filter `handled_by` → xem [0.0-modes-and-communication.md](./0.0-modes-and-communication.md).

Manager không phân biệt 2 path khi xử lý sub-agent — input + output giống nhau. Chỉ khác ở 2 hành vi đầu / đuôi turn:

- **Auto-greet (chỉ Mode 1-6)**: ngay khi user click card, Manager X tự sinh response chào + nêu scope nhóm (không qua sub-agent).
- **Compound question**: Manager X xử lý phần in-scope + flag phần off-scope cho user (không cross-call, không auto-route).

## Nguyên lý chung — Minimal interface

**Storage format ≠ Agent runtime input.**

| Layer | Format | Mục đích |
|---|---|---|
| **Storage** (bảng `messages`) | Đầy đủ: `user_message`, `conversation_id`, `user_id`, `metadata` (handled_by, last_search, ...), `role`, `content`, `created_at` | Lưu trữ, audit, query lịch sử |
| **Agent runtime input** | **Pure data — KHÔNG có ID, KHÔNG có context bundle generic.** Mỗi agent chỉ nhận đúng field nó dùng. | Để agent là pure function, dễ test, không phụ thuộc DB |

→ **Backend layer (orchestrator)** chịu trách nhiệm:
1. Pre-fetch DB (profile, memory, recent messages filtered, last_search_filter...)
2. Build input đúng schema của agent đó (chỉ field cần)
3. Gọi agent, nhận output (pure dict)
4. Persist output vào `messages` + state updates

Agent KHÔNG biết `conversation_id`, `user_id`, schema DB. Agent là **pure function input → output**.

## Input contract per agent (minimal)

Mỗi agent (Manager / sub-agent / Supervisor) có input contract riêng, chỉ field dùng:

| Agent | Input contract |
|---|---|
| **Supervisor** (Mode 0) | `user_message` + `recent_user_msgs[]` (mỗi msg có content + `handled_by` tag) |
| **Manager X** | `user_message` + những field profile/memory **X dùng cho dispatch** + những field cần để pass xuống sub-agent + `last_search_filter` (chỉ C) + `recent_messages[]` của X (đã filter `handled_by=manager_x`) |
| **Sub-agent** | Chỉ field nó dùng — xem input contract trong doc từng sub-agent |
| **Memory** (Agent 2) | `user_message` + `recent_messages_full` + existing facts (filter category) |
| **Profile** (Agent 3) | `user_message` + `recent_messages_full` + current profile row |

→ **KHÔNG pass `conversation_id`, `user_id`, snapshot toàn bộ profile/memory** cho agent. Backend pre-fetch đúng.

Output mỗi agent cũng pure dict — nếu cần state updates, agent trả structured dict, backend ghi DB.

## Cách hoạt động chung

### 1. Nhận input (Manager X runtime)

Backend đã pre-fetch & build input pure cho Manager X. Vd Manager B input:

```python
{
  "user_message": "Tôi học CNTT, muốn chuyển sang DA, nên học gì?",
  "profile": {
    "target_role": None,
    "current_role": "Sinh viên",
    "major": "CNTT",
    "mbti_type": None,
    "hard_skills": ["Python cơ bản"],
    # ... chỉ field Manager B + sub-agent B dùng
  },
  "memory_facts": [
    {"category": "preference", "content": "ngại commute xa"},
    # ... đã filter category cần
  ],
  "recent_messages": [...],  # đã filter handled_by=manager_b
}
```

KHÔNG có `conversation_id`, `user_id`, hay full DB snapshot.

### 2. Plan dispatch

LLM function calling với N function (= N sub-agent trong nhóm + 1 ask_back function).

Ví dụ Manager B có 4 function:
```
call_career_advisor:  Phân tích nghề phù hợp với profile user
call_skill_gap:       So sánh skill user vs target role
call_learning_path:   Đề xuất khoá học cho missing skills
ask_back_off_group:   Câu hỏi không thuộc tư vấn nghề
```

### 3. Chain trong nhóm (khi cần)

Manager tự chain sub-agent theo plan ban đầu, không hỏi user xác nhận từng bước:

```
plan = [career_advisor, skill_gap, learning_path]
out_1 = call_career_advisor(input)
out_2 = call_skill_gap(input + out_1)
out_3 = call_learning_path(input + out_2)
```

### 4. Synthesize output

Manager dùng LLM nhỏ tổng hợp output từ các sub-agent thành 1 response markdown cohesive (có heading, bố cục logic).

KHÔNG concat raw string từ từng agent.

### 5. Ask back nếu off-group

Nếu Manager A nhận câu hỏi **hoàn toàn** off-scope → ask back template scope-only:
> "Nhóm Khám phá bản thân hỗ trợ: MBTI / Holland / Goal setting.
>  Câu hỏi của bạn có vẻ về tìm việc — bạn muốn chuyển sang nhóm Tìm việc làm không?"

Cross-suggest group khác **case-by-case** — chỉ suggest khi Manager A biết chắc group đích có scope. Không có rule chung "always cross-suggest" hay "never cross-suggest" cho cả 6 Manager.

### 6. Compound question — xử lý in-scope + flag off-scope

Nếu message chứa **≥1 phần in-scope** và **≥1 phần off-scope** → Manager xử lý phần in-scope bằng sub-agent bình thường, đồng thời append note ở cuối response cho phần off-scope:

```
User trong Mode B: "Lộ trình học DA + lương DA bao nhiêu?"
  → Manager B:
    - In-scope (B): lộ trình học DA → call Learning Path
    - Off-scope (D): lương DA → flag
  → Response:
    [Learning Path output]
    "Riêng câu hỏi về lương DA, đây là Phân tích thị trường (nhóm D).
     Mình không xử lý ở nhóm này — bạn ghé nhóm D nhé."
```

KHÔNG cross-call agent nhóm D. KHÔNG auto-route. Chỉ text suggestion. Chi tiết → [0.0-modes-and-communication.md](./0.0-modes-and-communication.md).

### 7. Auto-greet khi user vào Mode 1-6

Khi user click card → Manager X tự sinh greet + nêu scope:
```
"Xin chào! Mình là [tên nhóm X].
 Nhóm này hỗ trợ:
  • [bullet 1] • [bullet 2] • [bullet 3]
 Bạn muốn bắt đầu với gì?"
```

Auto-greet không qua sub-agent. Mode 0 không auto-greet.

## Dispatch logic + chain pattern 6 nhóm

| Manager | Sub-agent | Single thường gặp | Chain thường gặp |
|---|---|---|---|
| **A · Khám phá** | Goal Setting, Assessment | Single | Hiếm chain |
| **B · Tư vấn nghề** | Career Advisor, Skill Gap, Learning Path | Single | Chain 3-bước (CA → SG → LP) khi user hỏi phức hợp |
| **C · Tìm việc** | Job Search, Company | Single | Optional: Company → Job Search |
| **D · Phân tích** | Market Insight, Salary | Single | Optional song song khi hỏi tổng quan ngành |
| **E · Tối ưu hồ sơ** | CV Parser, Resume Builder, Cover Letter | Theo workflow upload/generate | CV Parser → Resume Builder; Resume Builder → Cover Letter |
| **F · Phỏng vấn** | Question Gen, Mock Interview, Negotiation | Single | Mock Interview ngầm gọi Question Gen làm sub-tool |

## Ví dụ chain ở Manager B

```
User (free chat): "Tôi học CNTT, muốn chuyển sang Data Analyst, nên học gì?"
  → Supervisor classify → manager_B
  → Manager B nhận

Manager B (plan):
  1. call_career_advisor(user_message=..., profile_subset={target, mbti, edu, ...}, memory_preferences=[...])
     → {fit_score: 0.85, similar_occupations: [...], rationale}
  2. call_skill_gap(mode='by_role', target='Data Analyst',
                   profile_skills={hard_skills, soft_skills, certificates},
                   recent_messages=[...])
     → {missing: [SQL, Python, Tableau], category_grouped: {...}}
  3. call_learning_path(skill_list=[SQL, Python, Tableau],
                       profile_subset={years_exp, current_role, target_role, current_skills})
     → {ordered_courses: [...]}

(Lưu ý: KHÔNG truyền user_id / conversation_id xuống sub-agent — chỉ data pure.)

Manager B (synthesize):
  response_md = """
  ## 1. Đánh giá phù hợp
  Data Analyst phù hợp với background CNTT của bạn vì...

  ## 2. Skill còn thiếu
  - Hard: SQL, Python, Tableau
  - Tool: Excel nâng cao

  ## 3. Lộ trình học đề xuất
  Tuần 1-4: SQL Cơ bản (Coursera)
  Tuần 5-8: Python for Data Analysis (Udemy)
  ...
  """
```

## Không làm

- Manager KHÔNG cross-call agent của nhóm khác (vd Manager B không gọi Assessment của nhóm A). Nếu cần data nhóm khác → đọc từ DB (vd `user_assessment_results`).
- Manager KHÔNG re-route lên Supervisor.

Chi tiết logic dispatch / chain / synthesis → handle qua prompt engineering của từng Manager ở pha implement.
